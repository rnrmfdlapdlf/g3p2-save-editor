# 기본 저장 규칙과 공통 영역

## 값 저장 방식

현재 확인된 주요 정수값은 little-endian으로 저장되고, 실제 값은 저장값의 역수로 계산한다.

| 크기 | 읽기 규칙 | 쓰기 규칙 |
| --- | --- | --- |
| 2바이트 | `0xffff - readUint16LE(offset)` | `writeUint16LE(0xffff - value)` |
| 4바이트 | `0xffffffff - readUint32LE(offset)` | `writeUint32LE(0xffffffff - value)` |

체크섬 자체는 역수 저장값이 아니라 일반 2바이트 little-endian 값으로 읽고 쓴다.

## 세이브 타입과 에피소드 판별

| 오프셋 | 크기 | 의미 |
| --- | --- | --- |
| `0x08` | 4 | 장면 ID |
| `0x0c` | 4 | 세이브 타입 코드 |

세이브 타입 코드는 현재 다음 값으로 표시한다.

| 코드 | 표시 |
| --- | --- |
| `1` | 전투 |
| `4` | 챕터 |
| `7` | 연대표 |

전투 세이브의 에피소드 판별은 파티 인원보다 전투 인벤토리 count의 유효성을 먼저 본다.
EP4 전투와 EP5 전투는 모두 파티 테이블이 비어 있거나 다른 에피소드 테이블을 같이 가질 수 있기 때문이다.

## 플레이 타임

| 오프셋 | 크기 | 규칙 |
| --- | --- | --- |
| `0x04` | 3 | `0xffffff - 저장값`을 밀리초로 계산 |

## 체크섬

체크섬은 파일 마지막 2바이트에 저장된다.
계산 대상은 마지막 2바이트를 제외한 모든 바이트다.

```text
checksum = sum(data[i] * [3, 5, 7, 9][i % 4]) % 0x7d00
```

## 소지금

| 범위 | 오프셋 | 설명 |
| --- | --- | --- |
| EP4 챕터 | `0x63f8` | 영혼의 검 소지금 |
| EP5 챕터 | `0x74a8` | 뫼비우스의 우주 소지금 |

전투 소지금은 전투 인벤토리 count 바로 뒤 4바이트에 있다.

## 인벤토리

챕터 인벤토리 슬롯은 `아이템 코드 4바이트 + 수량 4바이트`가 반복된다.
count는 실제 저장된 슬롯 수다.

| 범위 | count | slots 시작 |
| --- | --- | --- |
| EP4 챕터 | `0x63f4` | `0x63fc` |
| EP5 챕터 | `0x74a4` | `0x74ac` |

현재 고정 수량으로 별도 정의한 소모품 코드는 다음과 같다.

| key | 이름 | 코드 |
| --- | --- | --- |
| `lowCapsule` | 로우캡슐 | `122` |
| `recoveryCapsule` | 회복캡슐 | `123` |
| `fullRecoveryCapsule` | 완전회복캡슐 | `124` |
| `stabilizer` | 안정제 | `125` |
| `goddessHolyWater` | 여신의 성수 | `126` |
| `mediumCapsule` | 미디엄캡슐 | `222` |
| `highCapsule` | 하이캡슐 | `223` |
| `bigCapsule` | 빅캡슐 | `224` |
| `gift` | 기프트 | `225` |

## 전투 인벤토리와 소지금

전투 세이브는 챕터 인벤토리와 다른 테이블을 쓴다.
소지금은 count 뒤 4바이트이며, slots는 소지금 뒤에서 시작한다.

| 범위 | count | money | slots 시작 |
| --- | --- | --- | --- |
| EP4 전투 | `0xad9b` | `0xad9f` | `0xada3` |
| EP5 전투 | `0xbe41` | `0xbe45` | `0xbe49` |
| EP4 전투 파일 안의 EP5 복사본 | `0xbe4b` | `0xbe4f` | `0xbe53` |

EP4 전투 파일에서 EP5 탭을 표시할 때는 실제 active episode가 EP4면 `episode5InEpisode4Battle` 테이블을 사용한다.

## 파티

파티 테이블은 `인원 수 4바이트 + 캐릭터 코드 4바이트 배열` 구조다.
현재 UI의 챕터 파티 편집 한도는 55명이다.

| 범위 | 오프셋 | 표시명 |
| --- | --- | --- |
| EP4 챕터 | `0x62f0` | 영혼의 검 |
| EP5 챕터 | `0x73a0` | 뫼비우스의 우주 |

전투 탭은 이 파티 테이블이 아니라 실제 전투 출격 유닛 스캔 결과를 표시한다.
